## 2. 核心模块：陪伴决策引擎

> 这是整个产品最重要的模块。

### 2.1 决策流程

```
定时器触发（每 5 分钟）
          │
          ▼
    ┌─────────────┐
    │  前置过滤器  │
    │             │
    │ □ 全屏中？    │──是──→ 跳过，退到托盘
    │ □ 免打扰？    │──是──→ 跳过
    │ □ 冷却期内？  │──是──→ 跳过
    │ □ 功能关闭？  │──是──→ 跳过
    │ □ 深夜+不活跃？│──是──→ 跳过
    └──────┬──────┘
           │ 全部通过
           ▼
    ┌─────────────┐
    │  触发规则检查 │  ← 先检查硬规则，不用每次都调 LLM
    │             │
    │ 规则 1: 早安问候
    │   条件: 今天第一次 + 7-10点
    │
    │ 规则 2: 长时间工作
    │   条件: 同一活动 > 2小时
    │
    │ 规则 3: 深夜提醒
    │   条件: 23-02点 + 用户活跃
    │
    │ 规则 4: 空闲超时
    │   条件: 空闲 > 30分钟
    │
    │ 规则 5: 提醒到期
    │   条件: reminders 表有到期项
    │
    │ 规则 6: 活动切换
    │   条件: 活动类型刚变化
    │
    │ 都没命中 → 跳过（省 token）
    └──────┬──────┘
           │ 某条规则命中
           ▼
    ┌─────────────┐
    │  消息生成    │
    │             │
    │ Level 1: 固定模板（0 token）
    │   早安/晚安/简单提醒
    │
    │ Level 2: 模板+记忆填充（0 token）
    │   "你昨天说的{topic}搞定了吗？"
    │
    │ Level 3: LLM 生成（消耗 token）
    │   复杂场景才用
    │
    │ 目标：70% 用 Level 1-2
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │  气泡展示    │
    │             │
    │ 头像→speaking │
    │ 气泡淡入      │
    │ 停留 15-60秒  │
    │ 自动淡出      │
    │ 记录说话时间   │
    └─────────────┘
```

### 2.2 消息生成策略（省 token）

| 级别 | 方式 | Token 消耗 | 使用场景 |
|------|------|-----------|---------|
| Level 1 | 固定模板 | 0 | 早安晚安、时间提醒 |
| Level 2 | 模板+记忆填充 | 0 | 结合记忆的问候 |
| Level 3 | LLM 生成 | 消耗 token | 复杂对话、用户主动聊天 |

目标比例：70% Level 1-2，30% Level 3。月费控制在 10-20 元。

## 3. UI 层设计

### 3.1 陪伴体头像

**状态和表情：**
- idle → 静态表情，微微浮动（呼吸动画）
- thinking → 思考表情，轻微旋转
- speaking → 说话表情，嘴巴动画
- sleeping → 睡觉表情 zzz（深夜模式）

**交互：**
- 鼠标悬停 → 微微放大 + tooltip
- 左键点击 → 展开对话面板
- 右键点击 → 快捷菜单
- 拖动 → 移动到任意位置

### 3.2 气泡消息

- 从头像旁边淡入（200ms）
- 默认停留 20 秒
- 鼠标悬停时计时暂停
- 淡出消失（300ms）
- 点击"回复"→ 展开面板
- 点击"关闭"→ 消失+冷却期
- 自动适应位置，不超出屏幕

### 3.3 展开式对话面板

- 从头像位置"长出来"（有展开动画）
- 380x600px，不是全屏
- 聊完点 ✕ 或 Esc → 收起回到头像
- 对话自动保存

## 4. 感知层设计

### 4.1 活动感知器

```
后台线程，每 10 秒：
  GetForegroundWindow() → 窗口句柄
  → 进程名 (如 "Code.exe")
  → 窗口标题

分类规则（纯本地）：
  code/vscode/intellij/terminal → coding
  chrome + github/stackoverflow → learning
  chrome + youtube/bilibili     → video
  steam/epic/游戏进程           → gaming
  微信/qq/discord              → social
  word/excel/outlook           → office
  spotify/网易云               → music
  无操作 > 30分钟              → idle
  其他                         → other

隐私：
  - 标题只在本地匹配关键词
  - 不记录完整标题
  - 用户可关闭
```

### 4.2 空闲检测

- GetLastInputInfo() 获取最后输入时间
- 30 分钟无操作 → idle
- 2 小时无操作 → away

### 4.3 全屏检测

- GetWindowPlacement() 检查
- 全屏时：头像退到托盘，不弹气泡

## 5. 数据流：一次主动说话

```
后台定时器 (每5分钟)
    │
    ▼
前置过滤 → 不在全屏/免打扰/冷却期
    │
    ▼
收集上下文：
  time: "23:30"
  activity: "coding"
  duration: "3小时15分"
  recent_memory: "用户最近在赶deadline"
    │
    ▼
规则匹配 → 命中"长时间工作" + "深夜"
    │
    ▼
消息生成（Level 2: 模板+记忆）：
  "都十一点半了，写了三个多小时了，
   注意休息啊，别跟上次一样熬到两点 😴"
    │
    ▼
气泡展示 → 停留25秒 → 用户没理 → 淡出
    │
    ▼
记录说话时间 → 冷却2小时
```

## 6. 窗口管理策略

1. **头像**：始终置顶，但不遮挡全屏，可拖动，默认右下角
2. **气泡**：跟随头像，始终置顶，可交互
3. **面板**：不置顶，Esc 收起
4. **全屏**：检测到全屏 → 退到托盘 → 退出全屏 → 恢复
5. **多显示器**：记住位置，切换时跟随
