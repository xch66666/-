# 数据库设计

> 这份文档定义所有数据表的结构、字段含义和关系。
> 使用 SQLite，所有数据存储在单个本地文件中。

## 2. 表结构详细定义

### 2.1 sessions（对话会话）

每次"新建对话"创建一个 session，包含多条 messages。

```sql
CREATE TABLE sessions (
    id          TEXT PRIMARY KEY,           -- UUID, 如 "a1b2c3d4-..."
    title       TEXT DEFAULT '新对话',       -- 会话标题（自动从首条消息生成）
    created_at  INTEGER NOT NULL,           -- 创建时间 (Unix 时间戳，毫秒)
    updated_at  INTEGER NOT NULL,           -- 最后活动时间
    message_count INTEGER DEFAULT 0,        -- 消息计数（冗余字段，优化列表查询）
    summary     TEXT,                       -- 会话摘要（会话结束时由 LLM 生成）
    is_archived INTEGER DEFAULT 0           -- 是否已归档 (0=否, 1=是)
);

CREATE INDEX idx_sessions_updated ON sessions(updated_at DESC);
```

### 2.2 messages（消息）

存储每一条对话消息，包括用户消息和 AI 回复。

```sql
CREATE TABLE messages (
    id          TEXT PRIMARY KEY,           -- UUID
    session_id  TEXT NOT NULL,              -- 所属会话 ID
    role        TEXT NOT NULL,              -- 'user' | 'assistant' | 'system'
    content     TEXT NOT NULL,              -- 消息正文
    created_at  INTEGER NOT NULL,           -- 发送时间
    model       TEXT,                       -- AI 使用的模型
    tokens_used INTEGER,                    -- 消耗的 token 数
    metadata    TEXT,                       -- JSON 格式附加信息

    FOREIGN KEY (session_id) REFERENCES sessions(id)
);

CREATE INDEX idx_messages_session ON messages(session_id, created_at);
```

**metadata 字段示例（JSON）：**
```json
{
  "tool_calls": [
    { "name": "get_weather", "input": {"city": "北京"}, "output": "晴天，25°C" }
  ],
  "memory_refs": ["mem_001", "mem_003"],
  "activity_at_send": "coding",
  "tts_played": true
}
```

### 2.3 memories（长期记忆）

从对话中自动提取的关键信息，跨会话持久化。

```sql
CREATE TABLE memories (
    id              TEXT PRIMARY KEY,
    content         TEXT NOT NULL,          -- 记忆内容，如"用户喜欢喝拿铁"
    category        TEXT NOT NULL,          -- 分类（见下方枚举）
    importance      INTEGER DEFAULT 3,      -- 重要度 1-5（5 最重要）
    source_session  TEXT,                   -- 来源会话 ID
    source_message  TEXT,                   -- 来源消息 ID
    created_at      INTEGER NOT NULL,
    last_recalled_at INTEGER,               -- 最后被检索使用的时间
    recall_count    INTEGER DEFAULT 0,      -- 被检索使用的次数
    is_active       INTEGER DEFAULT 1,      -- 是否有效
    user_edited     INTEGER DEFAULT 0,      -- 用户是否手动编辑过
    tags            TEXT,                   -- 标签，逗号分隔

    FOREIGN KEY (source_session) REFERENCES sessions(id)
);

CREATE INDEX idx_memories_category ON memories(category, is_active);
CREATE INDEX idx_memories_importance ON memories(importance DESC);
```

**category 枚举值：**
| 值 | 含义 | 示例 |
|---|------|------|
| preference | 偏好 | "用户喜欢用暗色主题" |
| fact | 事实 | "用户住在北京" |
| habit | 习惯 | "用户通常晚上 10 点后比较空闲" |
| relationship | 关系 | "用户的女朋友叫小雨" |
| work | 工作 | "用户是前端开发工程师" |
| emotion | 情感 | "用户最近压力比较大" |
| interest | 兴趣 | "用户在学 Rust 语言" |

**记忆淘汰策略：**
当记忆总数 > 500 条时，按以下公式打分淘汰：
```
score = importance * 0.4
      + recency_score * 0.3
      + recall_frequency * 0.2
      + (user_edited ? 1 : 0) * 0.1

删除 score 最低的 10% 记忆。
```

### 2.4 config（用户配置）

键值对形式存储所有用户设置。

```sql
CREATE TABLE config (
    key     TEXT PRIMARY KEY,
    value   TEXT NOT NULL,          -- JSON 字符串
    updated_at INTEGER NOT NULL
);
```

**配置键值清单：**

| key | value 示例 | 说明 |
|-----|-----------|------|
| persona | `{"name":"小伴","personality":"温柔","style":"口语化","nickname":"主人"}` | AI 人设 |
| tts | `{"enabled":true,"voice":"zh-CN-XiaoxiaoNeural","rate":1.0,"volume":0.8}` | 语音设置 |
| proactive | `{"enabled":true,"quiet_start":"23:00","quiet_end":"08:00"}` | 主动聊天 |
| activity | `{"enabled":true,"poll_interval_seconds":10}` | 活动感知 |
| appearance | `{"theme":"dark","accent_color":"#6366f1"}` | 外观 |
| llm | `{"provider":"openai","model":"gpt-4o-mini","max_context_messages":20}` | LLM 设置 |

### 2.5 activity_log（活动日志）

```sql
CREATE TABLE activity_log (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    activity    TEXT NOT NULL,              -- coding/learning/entertainment/social/work/idle
    window_title TEXT,
    process_name TEXT,
    started_at  INTEGER NOT NULL,
    ended_at    INTEGER,
    duration_seconds INTEGER
);

CREATE INDEX idx_activity_time ON activity_log(started_at DESC);
```

### 2.6 task_log（任务执行日志）

```sql
CREATE TABLE task_log (
    id          TEXT PRIMARY KEY,
    session_id  TEXT,
    message_id  TEXT,
    task_type   TEXT NOT NULL,              -- file_create/reminder/query/translate
    permission_level INTEGER NOT NULL,      -- 0/1/2
    user_approved INTEGER,                  -- NULL=未问, 0=拒绝, 1=批准
    input       TEXT NOT NULL,              -- JSON
    output      TEXT,                       -- JSON
    status      TEXT NOT NULL,              -- pending/approved/rejected/completed/failed
    error_message TEXT,
    created_at  INTEGER NOT NULL,
    completed_at INTEGER,

    FOREIGN KEY (session_id) REFERENCES sessions(id)
);
```

### 2.7 reminders（提醒事项）

```sql
CREATE TABLE reminders (
    id          TEXT PRIMARY KEY,
    content     TEXT NOT NULL,
    remind_at   INTEGER NOT NULL,
    is_recurring INTEGER DEFAULT 0,
    recurring_rule TEXT,
    is_done     INTEGER DEFAULT 0,
    source_session TEXT,
    created_at  INTEGER NOT NULL,

    FOREIGN KEY (source_session) REFERENCES sessions(id)
);

CREATE INDEX idx_reminders_time ON reminders(remind_at, is_done);
```

## 3. 数据生命周期

| 数据 | 创建时机 | 清理规则 |
|------|----------|----------|
| session | 用户点击"新建对话" | 用户手动归档/删除 |
| message | 每次发送/收到消息 | 随 session 删除 |
| memory | 对话结束后后台提取 | 超500条时自动淘汰低分 |
| config | 应用首次启动插入默认值 | 不清理 |
| activity_log | 活动类型变化时 | 保留最近30天 |
| task_log | AI 请求执行任务时 | 保留最近90天 |
| reminder | 用户通过对话设置提醒 | 已完成+过期的自动清理 |
