# AI 协作开发指南

> **这是最重要的文档。** 它告诉你如何高效地用 AI 帮你写代码、
> 如何组织提示词、遇到问题怎么办。

---

## 1. 核心原则

### 你不是在"让 AI 写一个软件"
### 你是在"当产品经理，管理 AI 这个程序员"

你的角色：决定做什么 / 定义功能细节 / 验收和测试 / 反馈修改方向
AI 的角色：负责写代码 / 实现功能 / 解答技术问题 / 修复 bug

### 黄金法则

1. **一次只做一个模块** -- 永远不要让 AI 一次写整个项目
2. **先跑通再优化** -- 丑但能用 > 漂亮但跑不起来
3. **给 AI 看上下文** -- 它看不到你的文件，你得告诉它现有代码长什么样
4. **小步验证** -- 每加一个功能就运行一次
5. **记录一切** -- 你做的每个决定、踩的每个坑都记下来

---

## 2. 推荐的 AI 工具搭配

- **Claude Code (终端)**: 适合文件操作、多文件修改、完整功能开发。能直接读写文件，理解项目结构
- **Cursor / Windsurf (编辑器)**: 适合边写边看、可视化编辑。IDE 集成，实时补全
- **Claude.ai 网页版**: 适合前期规划、讨论方案。长上下文，适合复杂讨论

建议：方案 A 或 B 做主力，方案 C 做辅助

## 3. 项目启动提示词（Phase 0）

### 3.1 环境搭建

把下面这段话发给 AI：

---
我是一个没有开发经验的人，要在 Windows 11 上开发一个 Tauri 2.0 桌面应用。
请帮我从零搭建开发环境。

我需要你：
1. 告诉我需要安装哪些工具，按顺序列出
2. 每个工具给我安装命令或下载链接
3. 安装完每个工具后告诉我如何验证安装成功
4. 最后帮我用 pnpm create tauri-app 创建项目，使用 React + TypeScript + Tailwind CSS
5. 项目名称：companion-app
6. 创建完后告诉我如何启动，让我看到一个空白窗口

注意：我可能遇到各种环境问题，请预判常见错误并给出解决方案。
一步一步来，每完成一步等我确认后再进行下一步。
---

### 3.2 了解项目结构

环境搭好后发给 AI：

---
我的 Tauri + React 项目已经创建好了。
请帮我理解这个项目的结构：

1. 列出项目的所有重要文件和文件夹，解释每个的作用
2. 告诉我前端代码在哪里（React 部分）
3. 告诉我后端代码在哪里（Rust 部分）
4. 前端和后端是怎么通信的（IPC 是什么）
5. 如何添加一个新的页面
6. 如何添加一个新的 Rust 命令

用非程序员能理解的方式解释。

## 4. 分模块开发提示词（Phase 1 - 核心对话）

### 4.1 第一步：让后端能调通 AI API

---
现在项目已经搭建好了。我要实现第一个功能：让 Rust 后端能调用 OpenAI API。

【现有项目结构】
（把 ls src-tauri/src/ 的结果粘贴给 AI）

【需求】
1. 在 Cargo.toml 中添加必要的依赖：
   - HTTP 客户端（推荐 reqwest）
   - JSON 处理（serde, serde_json）
   - 异步运行时（tokio）

2. 创建 src-tauri/src/ai/ 目录，实现 OpenAI API 客户端：
   - 支持发送消息并接收流式回复
   - API Key 从环境变量读取（先硬编码测试，后续改配置）
   - 模型默认使用 gpt-4o-mini

3. 创建一个简单的 IPC 命令 test:chat：
   - 输入：一段文字
   - 输出：AI 的回复（非流式，先简单跑通）

4. 在前端添加一个测试按钮，点击后调用 test:chat 并显示结果

请先实现最简版本，能跑通就行，不需要流式输出。
每一步都告诉我你在做什么，改了什么文件。
---

### 4.2 第二步：加上流式输出

---
上一步的 API 调用已经跑通了，AI 能回复消息。
现在我要加流式输出（streaming），让回复一个字一个字显示。

【需要修改的文件】
（把上一步改过的文件内容粘贴给 AI，或让它读取）

【需求】
1. 修改 AI 客户端，使用 OpenAI 的 streaming API
2. 每收到一个 chunk，通过 Tauri 的 emit 事件推送到前端
3. 事件名：chat:stream_chunk
4. 事件数据：{ content: string }
5. 流结束时发送 chat:stream_done 事件

前端侧：
6. 用 listen() 监听 chat:stream_chunk 事件
7. 每收到 chunk 就追加到显示区域
8. 监听 chat:stream_done，标记回复完成

请修改现有代码，不要重写。告诉我每处修改的位置和原因。
---

### 4.3 第三步：做聊天界面

---
流式输出已经可以工作了。现在我要做正式的聊天界面。

【需求】
创建以下 React 组件：

1. ChatWindow（主聊天窗口）
   - 顶部：会话标题 + 设置按钮
   - 中间：消息列表（可滚动，自动滚到底部）
   - 底部：输入框 + 发送按钮

2. MessageBubble（消息气泡）
   - 用户消息：靠右，蓝色背景
   - AI 消息：靠左，灰色背景
   - 显示时间戳
   - AI 消息支持 Markdown 渲染（代码块要有语法高亮）

3. StreamingIndicator（正在输入指示器）
   - AI 回复时显示跳动的三个点

【样式要求】
- 使用 Tailwind CSS
- 整体暗色主题
- 参考微信/Telegram 的聊天界面风格
- 输入框支持 Shift+Enter 换行，Enter 发送
- 发送按钮在输入为空时禁用
---

### 4.4 第四步：数据库持久化

---
聊天界面和 AI 回复都已经可以工作了，但关闭应用后消息就丢了。
现在要加上数据库持久化。

【数据库设计】（把 05-数据库设计.md 的 sessions 和 messages 表结构粘贴给 AI）

【需求】
1. 添加 Tauri SQLite 插件依赖
2. 在应用启动时执行建表 SQL（如果表不存在）
3. 实现 IPC 命令：
   - chat:create_session -> 创建新会话
   - chat:list_sessions -> 获取会话列表
   - chat:get_messages -> 获取消息（支持分页）
   - chat:send -> 发消息时同时保存到数据库
   - chat:delete_session -> 删除会话及其消息

4. 修改 chat:send：
   - 发送前保存用户消息到 DB
   - AI 回复完成后保存回复到 DB

5. 前端：添加会话列表侧栏，可切换/新建会话

数据库文件位置：~/.companion/companion.db

## 5. 分模块开发提示词（Phase 2 - 记忆系统）

### 5.1 短期记忆

---
现在对话功能已经完整了。接下来实现短期记忆。

【需求】
修改 chat:send 的后端逻辑，在调用 LLM 前组装完整 prompt：

1. 从数据库获取当前会话最近 N 条消息（默认 20）
2. 如果超过 N 条：前面的用 LLM 压缩为摘要
3. 组装消息列表：
   [system: 人设] + [system: 摘要] + [最近N条] + [新消息]

4. 逻辑放在 src-tauri/src/ai/prompt_builder.rs
---

### 5.2 长期记忆

---
短期记忆已完成。现在实现长期记忆。

【需求】
存储端：对话结束时用 LLM 提取关键信息写入 memories 表
检索端：新对话时检索相关记忆注入 prompt
前端：记忆管理页面（查看/编辑/删除/统计）

## 7. 分模块开发提示词（Phase 3 - 活动感知）

---
实现活动感知功能。

1. 后台线程每10秒轮询 GetForegroundWindow() + GetWindowTextW()
2. 活动分类器（纯本地规则匹配）：
   - code/vscode/terminal -> coding

3. 状态变化时 emit activity:changed 事件

## 10. 项目管理技巧

### 每次开发会话的标准流程

1. **开头**：告诉 AI 今天做什么 + 上次完成了什么
2. **提供上下文**：当前代码状态（粘贴或让 AI 读取文件）
3. **明确需求**：这次要做什么
4. **约定约束**：不要改动哪些文件、保持什么风格
5. **验收**：让 AI 告诉你如何测试
6. **记录**：让 AI 总结这次改了什么

### 用 CLAUDE.md 维护项目记忆

在项目根目录创建 CLAUDE.md，AI 每次启动时自动读取：
- 项目概述
- 已完成/未完成的功能列表
- 技术约定（命名规则、文件位置等）
- 当前开发重点
- 已知问题

### Git 提交节奏

每完成一个小功能就提交一次。让 AI 帮你写提交信息并提交。

## 11. 从"能跑"到"好用"的迭代路径

每个功能做完后，按顺序迭代：
1. "帮我检查有没有明显的 bug"
2. "用户体验有什么可以改进的？"
3. "review 一下这个模块的代码"
4. "有哪些边界情况没有处理？网络断开、超时、数据为空..."

---

## 12. 最后的建议

1. **不要追求一次完美** -- 先做出 60 分的，再优化到 80 分
2. **遇到不会的就问** -- AI 不会嫌你问题蠢
3. **保存好你的文档** -- docs 文件夹就是你的产品说明书
4. **做好心理准备** -- 遇到 bug 很正常
5. **享受过程** -- 看着自己做出的东西跑起来，很有成就感
