## Phase 0：环境搭建 + 项目初始化（3-5 天）

### 目标
一个能跑起来的 Tauri 空窗口 + 开发环境全部就绪。

### Step 1：安装开发工具（1-2 天）

需要安装的东西：
1. Node.js（LTS 版本）
2. Rust（通过 rustup）
3. pnpm（包管理器）
4. VS Code + 插件（Tauri, Rust Analyzer, Tailwind CSS IntelliSense）
5. Git

### Step 2：创建项目 + 跑通空窗口（1-2 天）

- 用 `pnpm create tauri-app` 创建项目
- 技术选择：React + TypeScript
- 配置 Tailwind CSS
- 运行 `pnpm tauri dev` 看到一个窗口
- 配置透明无边框窗口（为后续悬浮角色做准备）

### Step 3：理解项目结构（1 天）

- src-tauri/src/ = Rust 后端代码
- src/ = React 前端代码
- tauri.conf.json = Tauri 配置
- Cargo.toml = Rust 依赖
- package.json = 前端依赖

### 给 AI 的提示词
```
我要在 Windows 11 上开发一个 Tauri 2.0 桌面应用。
这个应用是一个桌面 AI 陪伴体，需要一个透明无边框的悬浮窗口。

请帮我：
1. 确认 Node.js, Rust, pnpm, Git 都已安装（给我验证命令）
2. 创建 Tauri 项目：React + TypeScript，项目名 companion-app
3. 配置 Tailwind CSS
4. 修改 tauri.conf.json：
   - decorations: false（无边框）
   - transparent: true（透明背景）
   - always_on_top: true（始终置顶）
   - 窗口大小 300x400
5. 运行项目，让我看到一个透明窗口

每一步等我确认后再继续。如果我遇到报错，请帮我分析解决。
```

### 产出
- 能运行的 Tauri 项目
- 透明无边框置顶窗口
- 所有工具安装完毕

## Phase 1：桌面存在感（约 2 周）

### 目标
桌面上有一个 Live2D 角色，能显示气泡消息，能展开对话面板。
**先不管智能不智能，先让它"存在"。**

### Step 1：简单头像占位（2-3 天）

> 先用简单圆形头像跑通流程，后面再换 Live2D。

- 100x100px 圆形头像组件
- 始终置顶，可拖动
- 轻微浮动/呼吸动画（CSS）
- 记住位置（保存到本地）
- 鼠标悬停放大

### Step 2：气泡消息组件（2-3 天）

- 从头像旁边弹出的气泡（带小三角指向头像）
- 支持文字内容
- 淡入动画（200ms）+ 淡出（300ms）
- 20 秒后自动消失
- "回复"按钮 + "关闭"按钮
- 鼠标悬停时计时暂停
- 位置自适应（不超出屏幕）

### Step 3：展开式对话面板（3-4 天）

- 点击头像或气泡"回复"按钮时展开
- 380x600px，不是全屏
- 消息列表 + 输入框 + 发送按钮
- 先用假数据（硬编码消息），不接 AI
- 有展开/收起动画
- Esc 键收起

### Step 4：系统托盘（1-2 天）

- 托盘图标
- 右键菜单：显示/隐藏、设置、退出
- 双击显示头像

### Step 5：替换为 Live2D 角色（3-4 天）

> 前面 4 步跑通后，把圆形头像换成 Live2D 角色。

- 安装 pixi.js + pixi-live2d-display
- 下载一个官方免费模型（推荐 Haru 或 Hiyori）
- 创建 Live2D 组件替换圆形头像
- 启用呼吸动画、眨眼动画
- 启用鼠标跟踪（角色眼睛跟随鼠标）
- 点击角色触发反应动画

## Phase 2：注入灵魂（约 2-3 周）

### 目标
角色开始知道你在干嘛，会主动说话，记得聊过什么。
**做完这一步，它从"桌面手办"变成"AI 陪伴"。**

### Step 1：活动感知（4-5 天）

- Rust 后端创建 activity 模块
- Win32 API：GetForegroundWindow + GetWindowTextW + 进程名
- 活动分类器（进程名/标题关键词 → 活动类型）
- 空闲检测：GetLastInputInfo（30 分钟无操作 → idle）
- 全屏检测：GetWindowPlacement
- 每 10 秒轮询，状态变化 emit 事件
- 记录到 activity_log 表

### Step 2：AI 对话接入（3-4 天）

- 后端创建 ai 模块，接入 OpenAI API（先用 gpt-4o-mini，便宜）
- 流式输出：SSE 解析 → emit chunk 事件 → 前端逐字显示
- 对话面板连接后端，能真正聊天
- SQLite 初始化（sessions + messages 表）
- 消息持久化

### Step 3：记忆系统（4-5 天）

- **短期记忆**：当前会话最近 20 条消息，超出部分 LLM 压缩为摘要
- **长期记忆**：对话结束（10 分钟无消息）→ LLM 提取关键信息 → 存 memories 表
- **记忆检索**：新对话首条消息 → 按重要度检索前 10 条记忆 → 注入 prompt
- **记忆管理界面**：展开面板中的记忆标签页（查看/编辑/删除）
- 淘汰机制（>500 条时自动清理低分记忆）

### Step 4：陪伴决策引擎（4-5 天）

- 定时器每 5 分钟触发评估
- **前置过滤器**：全屏? 免打扰? 冷却期? 深夜+不活跃? → 跳过
- **触发规则**：
  - 早安问候（今天首次 + 7-10 点）
  - 长时间工作（同一活动 > 2 小时）
  - 深夜提醒（23-02 点 + 用户活跃）
  - 空闲超时（idle > 30 分钟）
  - 提醒到期（reminders 表）
  - 活动切换（类型刚变化）
- **消息生成**：
  - Level 1：固定模板（0 token）
  - Level 2：模板 + 记忆填充（0 token）
  - Level 3：调 LLM 生成（消耗 token）
- 连接气泡组件展示

### Step 5：Live2D 表情映射（2-3 天）

- 陪伴体状态 → Live2D 表情/动作绑定：
  - idle → 默认表情 + 呼吸
  - speaking → 开心表情 + 说话动作
  - thinking → 歪头（LLM 生成回复时）
  - sleeping → 困倦表情（深夜）
  - concerned → 担心表情（长时间工作/深夜提醒）
- 气泡出现时自动切换 speaking 表情
- 气泡消失后恢复 idle

### Step 6：防打扰机制（1-2 天）

- 免打扰时段（可配置，默认 23:00-08:00）
- 冷却期（用户关闭气泡 → 2 小时内不再主动说话）
- 全屏自动退到托盘
- 设置页面：开关、时段、冷却时间

### 给 AI 的提示词

**Step 1（活动感知）：**
```
实现用户活动感知模块（Rust 后端）。

【需求】
1. 创建 src-tauri/src/activity/ 模块
2. 后台线程每 10 秒：
   - GetForegroundWindow() 获取前台窗口
   - 获取进程名和窗口标题
   - 分类为: coding/learning/browsing/video/gaming/social/office/music/idle/other
3. 空闲检测：GetLastInputInfo()，30分钟无操作 → idle
4. 全屏检测：GetWindowPlacement()，全屏时标记
5. 状态变化时 emit("activity:changed", { activity, duration })
6. 记录到 SQLite activity_log 表

【分类规则】
- code/vscode/cursor/terminal/cmd → coding
- chrome/firefox/edge（默认）→ browsing
- steam/epic → gaming
- 微信/qq/discord → social
- word/excel/outlook → office
- spotify/music → music
- 无操作>30min → idle
- 其他 → other

【注意】
- 所有处理在本地，不上传任何数据
- 只记录活动分类，不记录完整窗口标题
- 用户可通过 config 关闭此功能

需要添加什么 Cargo 依赖请告诉我。
```

**Step 4（决策引擎）：**
```
实现陪伴决策引擎。

【输入数据】（从其他模块获取）
- 当前活动类型 + 持续时间（from activity monitor）
- 当前时间（系统时钟）
- 用户状态：active/idle/away（from idle detector）
- 最近对话摘要（from memory）
- 待提醒事项（from reminders 表）
- 上次主动说话时间 + 冷却状态

【前置过滤】以下情况直接跳过：
- 全屏中
- 免打扰时段（config: proactive.quiet_start ~ quiet_end）
- 冷却期内（上次说话后 cooldown_minutes 内）
- 深夜 + 用户不活跃

【触发规则】按优先级检查：
1. 早安：今天首次评估 + 7-10点
2. 提醒到期：reminders 表有到期项
3. 长时间工作：同一活动 > 2小时
4. 深夜提醒：23-02点 + 用户活跃
5. 空闲超时：idle > 30分钟
6. 活动切换：类型刚变化

【消息生成】
- Level 1（固定模板）：早安/晚安 → 直接拼接字符串
- Level 2（模板+记忆）：从 memories 表取相关记忆填充模板
- Level 3（LLM 生成）：复杂场景才调 API

【输出】
返回 { should_speak, message, level, trigger_rule }
should_speak=true 时调用气泡组件显示消息。

每 5 分钟评估一次。请给出完整的代码结构。
```

### 产出
- AI 知道你在做什么
- AI 会在合适时机主动说话
- AI 记得你们聊过什么
- Live2D 角色会做表情
- 知道什么时候该闭嘴

## Phase 3：丰富体验（约 2 周）

### Step 1：人设系统（3-4 天）
- 设置面板（从头像展开）
- 配置：名字/性格/风格/称呼/自定义 prompt
- System Prompt 模板引擎
- 性格映射（温柔/活泼/毒舌/冷静/傲娇）
- 人设影响对话风格 + 主动说话风格

### Step 2：语音输出 TTS（3-4 天）
- Edge-TTS 集成（Rust 直接请求微软 TTS 接口，或用 Python 子进程）
- AI 说话时自动朗读（可关闭）
- 音色选择界面
- 语速/音量调节

### Step 3：任务执行（3-4 天）
- Function Calling 接入
- 权限三级制（Level 0 自动/Level 1 确认/Level 2 二次确认）
- 确认弹窗 UI
- 初始任务：创建笔记、设置提醒、查天气

### Step 4：打磨（3-4 天）
- 亮色/暗色主题
- 开机自启动（可选）
- 错误处理（网络断开、API 超时、数据库错误）
- 性能优化（内存、CPU）
- Live2D 形象选择（预设 + 自定义导入）
- 多显示器支持

### 产出
- 完整的人设配置
- 语音朗读
- 简单任务执行
- 整体打磨完成

## 关键原则

1. **Phase 1 先做出"存在感"** —— 哪怕什么都不会，先让它待在桌面上
2. **Phase 2 再注入"灵魂"** —— 感知+决策+记忆，从手办变成陪伴
3. **每完成一步就体验** —— 感受它从空壳到有灵魂的变化
4. **先能跑再好看** —— 丑但能用 > 漂亮但跑不起来
5. **一次一个模块** —— 不要让 AI 一次写太多，做完验证了再做下一个
6. **遇到报错别慌** —— 把完整报错复制给 AI，比你自己 Google 快
