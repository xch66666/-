# 产品需求文档 (PRD) - v2

> 核心理念：这不是一个聊天应用，而是一个住在你电脑里的陪伴。

---

## 1. 产品灵魂

**产品名称：** Companion（暂定）
**一句话定位：** 一个住在你桌面上的 AI 小伙伴

### 它是什么

```
它像一个室友，住在你的电脑里。
你忙的时候，它安静待在角落。
你闲的时候，它凑过来说两句。
它知道你在干嘛，知道你的习惯，记得你说过的事。
它不是你去找的工具，而是主动来找你的伙伴。
```

### 它不是什么

```
它不是 ChatGPT —— 不是你去打开、提问、关闭的应用
它不是豆包电话模式 —— 不是实时语音对话的通话
它不是桌面宠物 —— 不只是卖萌，它有真正的智能和记忆
它不是语音助手 —— 不是等你下命令的 Siri
```

## 2. 产品形态

### 2.1 核心形态：桌面悬浮陪伴体

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│                     你的桌面                              │
│                                                          │
│                                                          │
│                                                          │
│                                                          │
│                                         ┌──────────┐     │
│                                         │ (◕‿◕)    │     │  ← 陪伴体（小头像）
│                                         └──────────┘     │     常驻桌面角落
│                                                          │     可以拖动
│                                    ┌──────────────────┐  │
│                                    │ 写了好久了，      │  │  ← 气泡消息
│                                    │ 要不要喝杯水？ 💧 │  │     自动出现
│                                    └──────────────────┘  │     几秒后淡出
│                                                          │
│                                          系统托盘 [◕]    │  ← 托盘图标
└──────────────────────────────────────────────────────────┘
```

### 2.2 三种状态

```
状态 1：安静陪伴（默认）
┌────────┐
│ (◕‿◕)  │  一个小头像，待在屏幕角落
└────────┘  不发出声音，不弹消息
            只是在那里

状态 2：主动说话
┌──────────────────────────┐
│ (◕‿◕)  嘿，你在看什么呀？ │  头像旁边冒出气泡
└──────────────────────────┘  几秒后自动消失
                              用户可以点击回复

状态 3：展开对话（用户主动触发，或点击气泡）
┌─────────────────────┐
│  Companion     — □ ✕│
├─────────────────────┤
│                     │
│  (展开的侧面板)      │  从头像位置展开一个小面板
│  可以打字聊天        │  不是全屏窗口
│  可以看记忆          │  聊完可以收起
│  可以改设置          │  回到安静陪伴状态
│                     │
├─────────────────────┤
│ │ 输入消息...    │ ▶ ││
└─────────────────────┘
```

### 2.3 位置和行为

- 默认待在屏幕右下角（系统托盘上方）
- 用户可以拖动到任意位置
- 记住上次的位置
- 自动避开全屏窗口（全屏时退到托盘）
- 窗口最大化时不遮挡内容

## 3. 核心能力

### C1：活动感知（核心中的核心）

> 这是整个产品的基础。如果不知道用户在干嘛，就无法做到"适时陪伴"。

**感知方式：**
- 通过 Win32 API 获取当前前台窗口的标题和进程名
- 每 10 秒检测一次
- 所有判断在本地完成，不上传任何数据

**感知能力：**
| 能感知到的 | 怎么感知 | 例子 |
|-----------|---------|------|
| 用户在用什么软件 | 进程名 | VS Code, Chrome, 微信, Steam |
| 大概在看什么类型的内容 | 窗口标题关键词 | GitHub → 编程, B站 → 看视频 |
| 用户空闲了多久 | 无键鼠操作计时 | 空闲 30 分钟 |
| 当前时间段 | 系统时钟 | 凌晨 1 点 |
| 使用时长 | 累计同一活动的时间 | 连续编程 3 小时 |

**感知不到的（隐私红线）：**
- 不看屏幕截图
- 不读取文件内容
- 不监听键盘输入
- 不记录聊天内容
- 不上传任何数据到云端

### C2：主动陪伴（产品的灵魂）

> 不是用户来找 AI，而是 AI 来找用户。

**核心原则：像一个好室友，知道什么时候该说话，什么时候该闭嘴。**

**主动说话的时机：**

| 时机 | 说什么 | 示例 |
|------|--------|------|
| 早上首次开机 | 早安问候 | "早上好～今天有什么计划？" |
| 长时间工作后 | 建议休息 | "都写了 2 小时代码了，起来活动活动吧 💪" |
| 深夜还在 | 提醒休息 | "都一点了...明天不累吗 😴" |
| 切换到娱乐 | 轻松聊天 | "开始摸鱼了？在看什么好玩的？" |
| 空闲很久 | 打破沉默 | "发呆呢？要不要聊聊？" |
| 记忆中的事项到期 | 提醒 | "你昨天说今天要交报告的，写完了吗？" |
| 活动切换时 | 自然过渡 | "从编程切到学习了？换个脑子也挺好" |

**绝对不打扰的场景：**

| 场景 | 行为 |
|------|------|
| 全屏应用（游戏/视频/演示） | 完全安静，退到托盘 |
| 用户正在通话 | 不主动说话 |
| 深夜到早晨（可配置） | 除非用户活跃，否则不说话 |
| 用户刚忽略了一条消息 | 进入冷却期（默认 2 小时） |
| 用户手动开启免打扰 | 完全安静 |

**说话方式：**
- 以桌面气泡形式出现，不是弹窗
- 气泡显示 15-30 秒后自动淡出
- 用户可以点击气泡回复（展开对话面板）
- 用户可以点击关闭（触发冷却期）
- 用户可以忽略（什么都不做，气泡自动消失）

### C3：记忆系统

> 它记得你们之间发生过的事。

**短期记忆：**
- 当前对话的上下文
- 最近一次聊了什么

**长期记忆：**
- 你的习惯（"他一般 10 点后才开始写代码"）
- 你的偏好（"她喜欢喝拿铁"）
- 你的重要信息（"他女朋友叫小雨"）
- 你们的关系（"我是他最信任的 AI 伙伴"）
- 你的情绪变化（"最近压力好像挺大的"）

**记忆如何影响陪伴行为：**
```
没有记忆的 AI：
  "你已经工作 3 小时了，休息一下吧"  ← 通用建议

有记忆的 AI：
  "你昨天也加班到很晚呢，今天别太辛苦了"  ← 它记得
  "你上次说的那个 bug 修好了吗？"         ← 它关心
  "周末了！上次你说想去的那家店还去吗？"   ← 它把你当朋友
```

### C4：自然对话

> 想聊的时候随时能聊。

- 点击陪伴体 → 展开对话面板
- 可以打字聊天
- 后期支持语音输入
- 对话自然，像跟朋友发微信
- 对话内容会被记忆系统记住

### C5：人设系统

> 它是谁，由你决定。

**可配置项：**
- 名字（给它起个名字）
- 性格（温柔 / 活泼 / 毒舌 / 冷静 / 傲娇 / 自定义）
- 说话风格（简洁 / 啰嗦 / 口语化 / 文艺）
- 它怎么称呼你
- 头像 / 表情样式

**性格如何影响陪伴：**

| 性格 | 看到你加班 3 小时的反应 |
|------|----------------------|
| 温柔 | "辛苦了，要不要休息一下？我给你放首歌？ 🎵" |
| 活泼 | "哇你都写了 3 小时了！起来蹦跶两下！🏃" |
| 毒舌 | "又加班？你这效率是不是该反思一下了 😏" |
| 冷静 | "已经 3 小时了，建议休息 15 分钟再继续" |
| 傲娇 | "我才不是关心你呢...只是你这样效率很低啦" |

### C6：简单任务（后期）

> 它能帮你做点小事，但需要你允许。

- 设置提醒："3 点后提醒我喝水"
- 创建笔记："帮我把这个想法记下来"
- 查天气/时间等简单查询
- 所有操作都需要你确认

### C7：Live2D 形象系统

> 它不只是一个头像，是一个活的二次元角色。

**技术方案：** PixiJS + pixi-live2d-display
- 使用 Live2D Cubism 模型，支持呼吸、眨眼、表情切换
- 角色眼睛会跟随鼠标移动
- 点击角色会触发反应动画

**形象状态映射：**
| 陪伴体状态 | Live2D 表现 |
|-----------|------------|
| 安静陪伴 | 呼吸动画 + 偶尔眨眼 |
| 主动说话 | 开心表情 + 说话动作 |
| 思考中 | 微微歪头 |
| 深夜模式 | 困倦表情 + zzz |
| 关心用户 | 担心表情 + 轻微前倾 |
| 被点击 | 惊喜反应动画 |

**自定义形象：**
- 内置 2-3 个预设形象可选
- 支持用户导入自己的 Live2D 模型（.model3.json）
- 可调整角色大小和位置
- 可开关呼吸/眨眼/鼠标跟踪等动画

**性能影响：**
- 内存增加约 30-50MB（总计 < 80MB，可接受）
- CPU 增加约 1-3%（可接受）
- 全屏/最小化时自动暂停渲染

详见：docs/10-Live2D形象系统.md

## 4. 非功能需求

### 性能（极其重要）
| 指标 | 目标值 | 为什么重要 |
|------|--------|-----------|
| 内存占用 | < 80MB | 它是常驻的，不能占太多资源 |
| CPU 占用 | < 1% 空闲时 | 不能影响用户正常使用 |
| 启动时间 | < 2 秒 | 开机自启必须快 |
| 安装包大小 | < 15MB | 轻量感 |
| 气泡响应 | < 3 秒 | 从"决定说话"到"气泡出现" |

### 存在感设计
```
它应该让人觉得"有个小伙伴在陪我"
而不是"有个程序在监视我"

关键区别：
  监视感 → "检测到你在编程 3 小时，建议休息"  ← 冷冰冰
  陪伴感 → "写了好久了呢，要不要起来走走？"   ← 有温度
```

### 隐私
- 所有活动感知数据只在本地处理
- AI API 调用时只发送必要的上下文
- 不把窗口标题原文发给 AI（只发活动类型分类）
- 用户可以随时关闭活动感知
- 用户可以随时删除所有记忆
- 不截图，不录屏，不监听

## 5. 用户一天的典型体验

```
08:30  开机
       桌面角落出现小头像："早上好～今天周三，加油哦 ☀️"

08:31  打开 VS Code 开始写代码
       陪伴体安静待在角落，不说话

10:45  已经连续编程 2 小时 15 分钟
       气泡出现："写了一早上了，要不要喝杯水再继续？💧"
       用户点了一下气泡 → 它开心地说 "记得多喝水哦"
       气泡淡出

12:00  打开 Chrome 开始看 B 站
       气泡："午饭时间到了，先吃饭还是先看视频？🍜"
       用户没理它 → 气泡 15 秒后自动消失

14:30  回到 VS Code 继续写
       气泡："回来啦，上午那个 bug 搞定了吗？"

17:00  打开微信聊天
       陪伴体安静，不偷看聊天内容

19:00  打开 Steam 玩游戏
       全屏 → 陪伴体自动退到托盘，完全安静

23:30  退出游戏，回到桌面
       气泡："回来啦，今天玩得开心吗？不过快 12 点了哦... 🌙"

00:15  还在刷手机（电脑空闲）
       气泡："都 12 点多了...明天不用早起吗？😴"

00:20  关机
       下次开机会记得今天聊过什么
```

## 6. 功能优先级（重新排列）

| 优先级 | 功能 | 原因 |
|--------|------|------|
| P0 | 桌面悬浮 Live2D 角色 + 气泡 | 这是产品形态本身 |
| P0 | 活动感知 | 没有它就不是"陪伴"，只是"聊天" |
| P0 | 主动说话 + 时机判断 | 产品的灵魂 |
| P0 | 对话能力 | 基础能力 |
| P0 | 记忆系统 | 让陪伴有连续性，不像每次都重新认识 |
| P1 | 人设系统 + Live2D 表情映射 | 让它有个性，表情丰富 |
| P1 | 防打扰机制 | 知道什么时候闭嘴和知道什么时候说话一样重要 |
| P1 | 系统托盘 | 安静时的容身之处 |
| P2 | 语音输出 | 让陪伴更有温度 |
| P2 | 任务执行 | 锦上添花 |
| P3 | 语音输入 | 后期 |
| P3 | 外观自定义 | 后期 |

## 7. 与之前版本的区别

```
v1（之前）：
  形态 = 聊天窗口应用
  核心 = 对话
  用户主动找 AI
  AI 被动回复

v2（现在）：
  形态 = 桌面悬浮陪伴体
  核心 = 陪伴 + 感知
  AI 主动找用户
  对话是陪伴的一种方式，不是全部
```
