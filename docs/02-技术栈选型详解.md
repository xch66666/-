# 技术栈选型详解

> 这份文档告诉你：每个技术是什么、为什么选它、它在项目中负责什么。

---

## 技术栈总览

```
用户看到的界面 ──→ React + TypeScript + Tailwind CSS
                     │
桌面窗口管理 ──→ Tauri 2.0 (Rust)
                     │
AI 对话能力 ──→ Claude API / OpenAI API
                     │
语音合成 ──→ Edge-TTS (微软免费服务)
                     │
语音识别 ──→ Whisper (OpenAI 开源模型)
                     │
数据存储 ──→ SQLite (本地数据库)
                     │
活动感知 ──→ Win32 API (Windows 系统接口)
                     │
版本控制 ──→ Git + GitHub
```

## 逐一详解

### 1. Tauri 2.0 —— 桌面应用框架

**它是什么：**
一个用 Rust 编写的框架，能把网页（HTML/CSS/JS）包装成桌面应用。
类似于 Electron（VS Code 用的那个），但体积小 10 倍以上。

**为什么选它：**
| 对比项 | Tauri | Electron | WPF (C#) |
|--------|-------|----------|----------|
| 安装包大小 | ~5MB | ~80MB+ | ~20MB |
| 内存占用 | ~30MB | ~150MB+ | ~50MB |
| 前端技术 | 任意 Web 技术 | 任意 Web 技术 | XAML |
| 学习曲线 | 中等 | 低 | 高 |
| Win11 适配 | 好 | 好 | 原生 |
| 生态/社区 | 活跃增长中 | 成熟 | 成熟但老化 |

**它在项目中负责什么：**
- 创建和管理桌面窗口
- 系统托盘图标
- 前端和后端的通信桥梁（IPC）
- 文件系统访问
- 打包安装程序

**你需要了解的：**
- Tauri 项目有两个部分：`src-tauri/`（Rust 后端）和 `src/`（Web 前端）
- 前端通过 `invoke()` 函数调用后端 Rust 代码
- 后端通过 `emit()` 函数向前端推送事件

### 2. React + TypeScript —— 前端

**React 是什么：**
Facebook 开发的 JavaScript 库，用来构建用户界面。
核心思想：界面 = 组件的组合。

**TypeScript 是什么：**
给 JavaScript 加上类型检查。
好处：AI 生成的代码更不容易出错，你也能更快发现 bug。

**为什么选 React 而不是 Vue：**
- React 社区更大，AI 对 React 代码的理解更好
- Tauri 官方对 React 模板支持最好
- 组件生态更丰富

**你需要了解的：**
- 组件（Component）：界面的基本单元，一个按钮是一个组件，一个聊天框也是
- State（状态）：组件里的数据，数据变了界面自动更新
- Props（属性）：父组件传给子组件的数据
- Hook：在组件中使用状态和副作用的方式（useState, useEffect 等）

### 3. Tailwind CSS —— 样式

**它是什么：**
一个 CSS 框架，用预定义的类名直接写样式，不用手写 CSS 文件。

**为什么选它：**
- 写起来快，AI 也特别擅长生成 Tailwind 代码
- 不需要单独管理 CSS 文件
- 内置响应式设计和暗色模式支持

**示例：**
```html
<!-- 不用 Tailwind -->
<div style="padding: 16px; border-radius: 8px; background: white; box-shadow: ...">

<!-- 用 Tailwind -->
<div class="p-4 rounded-lg bg-white shadow-md">
```

### 4. SQLite —— 本地数据库

**它是什么：**
一个嵌入式数据库，不需要安装数据库服务器，数据存在一个本地文件里。

**为什么选它：**
- 零配置，不需要安装任何东西
- 数据就是一个 `.db` 文件，方便备份
- 性能足够（我们不处理百万级数据）
- Tauri 有成熟的 SQLite 插件

**它在项目中负责什么：**
- 存储所有对话记录
- 存储长期记忆
- 存储用户配置（人设、偏好）
- 存储活动日志

---

### 5. Claude API / OpenAI API —— AI 大脑

**它是什么：**
大语言模型的云服务接口。你发消息给它，它返回回复。

**选型建议：**
| 服务 | 优势 | 劣势 | 月费估算 |
|------|------|------|----------|
| Claude Sonnet | 对话质量高，中文好 | 价格稍贵 | 30-100 元 |
| OpenAI GPT-4o-mini | 便宜，速度快 | 人格一致性稍差 | 10-50 元 |
| 本地模型 (Ollama) | 完全免费，离线可用 | 质量明显不如云端 | 免费 |

**建议策略：**
- 开发阶段用 GPT-4o-mini（便宜）
- 正式使用切换到 Claude Sonnet（质量好）
- 可选：支持本地模型作为离线备选

**你需要了解的：**
- API Key：一串密钥，用来验证你的身份和计费
- Token：AI 处理文本的单位，越多越贵
- Streaming：流式返回，一个字一个字返回而不是等全部生成
- System Prompt：系统提示词，定义 AI 的行为和人设
- Function Calling：让 AI 可以"调用函数"来执行操作

---

### 6. Edge-TTS —— 语音合成

**它是什么：**
微软 Edge 浏览器内置的 TTS（文字转语音）服务，免费可用。

**为什么选它：**
- 免费
- 中文语音质量不错
- 支持多种音色（男/女/不同风格）
- 有 Python 库可以直接调用

**备选方案：**
- Azure TTS（付费，质量更高）
- 本地 TTS（如 Piper TTS，免费但质量一般）

---

### 7. Win32 API —— 活动感知

**它是什么：**
Windows 操作系统的底层接口，可以获取系统信息。

**在项目中的作用：**
- `GetForegroundWindow()`：获取当前最前面的窗口
- `GetWindowText()`：获取窗口标题
- 通过窗口标题判断用户在做什么

**你不需要了解细节：**
让 AI 帮你写 Rust 代码调用这些 API 即可。
