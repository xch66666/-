# 接口设计文档

> 这份文档定义前端和后端之间如何通信。
> Tauri 使用 IPC（进程间通信）连接前端（JS/TS）和后端（Rust）。

---

## 1. IPC 通信机制

```
前端 (React/TypeScript)          后端 (Rust)
       |                              |
       |  invoke(命令名, 参数)         |
       | ---------------------------> |
       |                              |  执行逻辑
       |  Promise<返回值>              |
       | <--------------------------- |
       |                              |
       |  emit(事件名, 数据)           |
       | <--------------------------- |  后端主动推送
       |  listen(事件名, callback)     |
       |                              |
```

**两种通信方式：**
1. **invoke（请求-响应）**：前端主动调用后端，等待返回结果
2. **emit/listen（事件推送）**：后端主动向前端推送数据（如流式输出、活动变化）

## 3. 接口详细定义

### 3.1 chat:create_session

```typescript
// 请求
invoke("chat:create_session")

// 响应
interface Session {
  id: string;           // "a1b2c3d4-e5f6-..."
  title: string;        // "新对话"
  created_at: number;   // 1704067200000
  updated_at: number;
  message_count: number;
  is_archived: boolean;
}
```

### 3.2 chat:list_sessions

```typescript
invoke("chat:list_sessions", {
  limit?: number;    // 默认 50
  offset?: number;   // 默认 0
})

// 响应: { sessions: Session[], total: number }
```

### 3.3 chat:get_messages

```typescript
invoke("chat:get_messages", {
  session_id: string;
  limit?: number;     // 默认 50
  before?: string;    // 在此消息ID之前的消息（翻页用）
})

interface Message {
  id: string;
  session_id: string;
  role: "user" | "assistant" | "system";
  content: string;
  created_at: number;
  model?: string;
  metadata?: {
    tool_calls?: ToolCall[];
    memory_refs?: string[];
    activity_at_send?: string;
    tts_played?: boolean;
  };
}

// 响应: { messages: Message[], has_more: boolean }
```

### 3.4 chat:send

```typescript
invoke("chat:send", {
  session_id: string;
  content: string;
})

// 立即返回: { message_id: string, session_id: string }
// 后续通过事件接收 AI 回复：
//   chat:stream_chunk -> { session_id, content: string }
//   chat:stream_done  -> { session_id, message: Message, memories_extracted: number }
//   chat:stream_error -> { session_id, error: string, retryable: boolean }
```

### 3.5 chat:stop

```typescript
invoke("chat:stop", { session_id: string })
// 响应: { stopped: boolean }
```

### 3.6 memory:list

```typescript
invoke("memory:list", {
  category?: string;
  limit?: number;    // 默认 100
  offset?: number;
})

interface Memory {
  id: string;
  content: string;       // "用户喜欢喝拿铁"
  category: string;      // "preference"
  importance: number;    // 3
  created_at: number;
  last_recalled_at: number | null;
  recall_count: number;
  user_edited: boolean;
  tags: string[];
}

// 响应: { memories: Memory[], total: number }
```

### 3.7 memory:update / memory:delete

```typescript
invoke("memory:update", {
  id: string;
  content?: string;
  importance?: number;
  tags?: string[];
})
// 响应: { success: boolean; memory: Memory }

invoke("memory:delete", { id: string })
// 响应: { success: boolean }
```

### 3.8 memory:stats

```typescript
invoke("memory:stats")
// 响应:
// { total, by_category: Record<string,number>,
//   max_capacity: 500, usage_percent, oldest_memory, newest_memory }
```

### 3.9 config 相关

```typescript
invoke("config:get", { key: string })
// 响应: { value: any }

invoke("config:set", { key: string, value: any })
// 响应: { success: boolean }
// 同时触发事件: config:changed -> { key, value }

invoke("config:get_all")
// 响应: 包含所有配置块的完整对象
```

### 3.10 配置类型定义

```typescript
interface PersonaConfig {
  name: string;           // AI 名字

interface TTSConfig {
  enabled: boolean;
  voice: string;          // 音色 ID

}

interface ProactiveConfig {
  enabled: boolean;

  quiet_end: string;      // "08:00"

}

interface LLMConfig {
  provider: "openai" | "anthropic" | "local";
  model: string;
  api_key_encrypted?: string;
  max_context_messages: number;
  base_url?: string;
}
```

### 3.11 任务确认流程

```
AI 返回 tool_use
       |
       v
后端判断权限等级
       |
       |-- Level 0 -> 自动执行 -> 结果返回 LLM
       |
       +-- Level 1/2 -> 触发事件:
           task:request_confirm -> {
             task_id, task_type, description,
             permission_level, details
           }

           前端弹出确认对话框
           |-- 用户确认 -> invoke("task:approve", { task_id })
           +-- 用户拒绝 -> invoke("task:reject", { task_id })
```

### 3.12 tts:speak / tts:list_voices

```typescript
invoke("tts:speak", { text: string; voice?: string; rate?: number })
// 响应: { speaking: boolean }
// 事件: tts:status_changed -> { status: "speaking"|"paused"|"stopped"|"done" }

invoke("tts:list_voices")
// 响应: Voice[]
// { id, name, gender, language, style? }
```

## 4. 错误处理规范

### 统一错误格式
```typescript
interface AppError {
  code: string;
  message: string;
  details?: any;
  retryable: boolean;
}
```

### 错误代码表
| code | 含义 | 处理建议 |
|------|------|----------|
| LLM_API_ERROR | AI API 调用失败 | 检查网络和 API Key |
| LLM_RATE_LIMIT | API 请求过于频繁 | 等待后重试 |
| LLM_TOKEN_LIMIT | 消息太长超出限制 | 减少上下文 |
| DB_ERROR | 数据库操作失败 | 重试或重启应用 |
| TTS_ERROR | 语音合成失败 | 检查网络 |
| PERMISSION_DENIED | 用户拒绝授权 | 正常流程 |
| FILE_NOT_FOUND | 文件不存在 | 提示用户 |
| NETWORK_ERROR | 网络错误 | 检查网络 |
| UNKNOWN | 未知错误 | 查看日志 |
